# Laravel 総合実践演習課題

## 前提条件

以下の内容を学習済みであることを前提とします。

- Laravel 13の基本構造を理解している
- Controller / Model / View / Migration の役割を理解している
- CRUD処理を実装した経験がある
- Laravel 13のStarter Kitを使った認証機能を学習済み
- auth middlewareによるログイン制限を理解している
- Policyによる認可制御を学習済み
- Git / GitHub を使ったソースコード管理の基本を理解している

---

# 実践演習課題

この課題は、Laravelの個別機能を学ぶための短い演習ではありません。

実務に近い要件をもとに、受講者自身が設計し、実装し、動作確認し、成果物としてまとめる総合演習です。

これまでのレッスンでは、次のような機能を個別に学んできました。

- 認証
- 認可
- CRUD
- バリデーション
- リレーション
- ファイルアップロード
- メール通知
- API
- テスト

本課題では、これらを組み合わせて「実際に動作するアプリケーション」を構築していただきます。

---

# 課題概要

| 項目 | 内容 |
|---|---|
| 対象 | Laravelの基礎学習を一通り終えた受講者 |
| 形式 | 個人での開発演習 |
| 想定時間 | 18時間 |
| 成果物 | 動作するLaravelアプリ、README、GitHubリポジトリ |
| 重視すること | 要件理解、設計意図、動作確認、説明力 |

---

# 学習目的

この演習では、以下を目的とします。

- Laravelの各機能を組み合わせて開発できるようになる
- 実務を意識した設計・実装を経験する
- 認証・認可を含むアプリ構築を理解する
- GitHubを用いた成果物管理に慣れる
- READMEによる説明力を身につける
- テストや動作確認を含めた品質意識を持つ

---

# 成果物に含めるもの

## 必須

- Laravelアプリケーション
- GitHubリポジトリ
- README.md

## READMEに記載する内容

- アプリ概要
- 使用技術
- ER図（任意）
- セットアップ方法
- 主な機能一覧
- ログイン方法（テストユーザー）
- 工夫した点
- 苦労した点
- 今後改善したい点

---

# 評価観点

## 1. 要件理解

- 必要な機能を整理できているか
- ログイン制御が適切か
- 認可が正しく実装されているか

## 2. 設計

- テーブル設計が適切か
- リレーション設計が自然か
- 責務分離ができているか

## 3. 実装

- CRUDが安定して動作するか
- バリデーションが適切か
- UIが最低限使いやすいか

## 4. コード品質

- 命名が適切か
- 可読性があるか
- 不要コードがないか

## 5. テスト・動作確認

- エラー時の挙動を確認しているか
- 想定通りに動作するか
- テストコードを書いているか

## 6. 説明力

- READMEが整理されているか
- セットアップ手順が分かりやすいか
- 実装意図を説明できるか

# 推奨開発フロー

1. 要件整理
2. 画面一覧作成
3. テーブル設計
4. ルーティング整理
5. Model / Migration作成
6. 認証実装
7. CRUD実装
8. 認可実装
9. バリデーション実装
10. テスト
11. README作成
12. GitHub提出

---

# 注意事項

- 完璧なUIデザインは不要
- まずは「正しく動くこと」を重視する
- 実装理由を説明できることが重要
- コピペではなく、自分で理解して実装すること
- エラー対応も学習の一部として取り組むこと

# 最終目標

Laravelの機能を「個別に知っている状態」から、「実際のアプリとして組み合わせて開発できる状態」になることを目指します。
