# 18時間の全体構成

この課題は、次の11レッスンで進めます。

| 回 | レッスン | 目的 |
|---|---|---|
| 1 | 課題概要・業務要件理解 | 何を作るか理解する |
| 2 | 要件整理・画面 / DB設計 | 画面とデータ構造を整理する |
| 3 | プロジェクト初期構築 | Laravel 13プロジェクトと認証を準備する |
| 4 | DB実装・初期データ準備 | Migration、Model、Seederを準備する |
| 5 | タスク管理CRUD実装 | 登録・一覧・詳細・編集・削除を実装する |
| 6 | 認証・認可・権限制御 | 自分のタスクだけ操作できるようにする |
| 7 | 業務向け拡張機能実装 | 検索、絞り込み、期限表示などを追加する |
| 8 | 動作確認・改善・品質向上 | 不具合修正と使いやすさを改善する |
| 9 | GitHub提出・提出準備 | コードをGitHubへ提出できる状態にする |
| 10 | セルフレビュー・改善対応 | セルフレビュー観点でコードを見直す |
| 11 | 成果発表・まとめ | 成果物を説明し、振り返る |

---

# 成果物として提出するもの

課題終了時には、次の成果物を提出します。

| 成果物 | 内容 |
|---|---|
| Laravelアプリ | 動作するソースコード一式 |
| GitHubリポジトリ | コード管理用 |
| .env.example | 環境変数のサンプル |
| README | 起動手順、機能一覧、使用技術、工夫点 |
| 発表資料または発表メモ | 任意。成果発表で説明する内容 |
| スクリーンショット | 任意。READMEや発表で利用 |
