# 業務タスク管理アプリ 要件定義

## アプリ概要

今回作成するアプリは、社内メンバーが自分の業務タスクを登録し、進捗を管理できる「業務タスク管理アプリ」です。

---

# 想定する利用シーン

社内メンバーが、日々の業務タスクを登録し、進捗や期限を管理します。

たとえば、次のようなタスクを扱います。

- 見積書を作成する
- 顧客へメールを返信する
- 会議資料を準備する
- 週次レポートを提出する
- 不具合調査を行う
- 社内申請を処理する

---

# 想定ユーザー

このアプリでは、主に次のユーザーを想定します。

| 役割 | 権限概要 |
|---|---|
| 一般ユーザー | 自分のタスクのみ登録・閲覧・編集・削除できる |
| 管理者 | 他ユーザーのタスクも閲覧・操作できる機能を発展要件として検討する |
| 未ログインユーザー | タスク管理画面にはアクセスできない |

---

# 必須要件一覧

この課題で必ず実装する機能です。

| 機能名 | 詳細 |
|---|---|
| ログイン / ログアウト | Laravel 13のStarter Kitを使ったWeb認証を利用する |
| タスク登録 | ログインユーザーが自分のタスクを登録できる |
| タスク一覧 | 自分のタスクを一覧で確認できる |
| タスク詳細 | 自分のタスクの詳細を確認できる |
| タスク編集 | 自分のタスクを編集できる |
| タスク削除 | 自分のタスクを削除できる |
| ステータス管理 | 未着手 / 進行中 / 完了 を管理できる |
| 担当者設定 | タスクに user_id を持たせ、ユーザーと紐づける |
| 認可制御 | 他人のタスクは編集・削除できないようにする |
| バリデーション | 必須項目、文字数、日付などを検証する |

---

# 推奨要件

必須要件が完成したあと、余裕があれば実装する機能です。

| 機能名 | 詳細 |
|---|---|
| 検索 | タスク名や説明で検索できる |
| ステータス絞り込み | 未着手、進行中、完了で絞り込める |
| 期限管理 | 期限日を設定できる |
| 期限切れ表示 | 期限を過ぎたタスクが分かる |
| 並び替え | 作成日順、期限日順などで並び替えできる |
| フラッシュメッセージ | 登録・更新・削除の結果を表示する |
| ページネーション | タスク数が多い場合にページ分割する |

---

# 発展要件

このレッスンでは実装しませんが、後半のレッスンで選択肢として扱います。

| 機能名 | 詳細 |
|---|---|
| 管理者ロール | 管理者だけが全タスクを閲覧できる |
| 管理者用画面 | 全ユーザーのタスクを管理できる |
| 添付ファイル | タスクに資料ファイルを添付できる |
| メール通知 | タスク登録時や期限前に通知する |
| API化 | タスク管理機能をAPIとして提供する |
| API認証 | SanctumでAPIを保護する |
| テスト | Feature Testで主要機能を検証する |
